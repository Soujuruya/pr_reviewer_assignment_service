### Запуск проекта

1. Настройка окружения

Скопируйте пример файла конфигурации и заполните его своими значениями:

```bash
cp configs/.env.example configs/.env
```

Файл configs/.env содержит настройки базы данных, порта сервера и логирования.

2. Сборка и запуск через Docker

С помощью Makefile можно управлять контейнерами и миграциями.

Сборка Docker-образов:

```bash
make build
```
Запуск всех сервисов c применением миграций(сервис готов к работе):

```bash
make up
```
Остановка и удаление контейнеров с томами:

```bash
make down
```

Просмотр логов:

```bash
make logs
```
3. Миграции базы данных

Применить все миграции:

```bash
make migrate-up
```

Откатить миграции:

```bash
make migrate-down
```

Проверить текущую версию миграции:

```bash
make make migrate-version
```

4. Локальный запуск без Docker

Для разработки можно запускать сервис напрямую с локальным Go:

```bash
make run-local
```
